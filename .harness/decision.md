## 2026-04-30: TA 3건 코드+VERIFY 동시 fix — VERIFY는 코드 검증 강도의 거울

- **선택**: VERIFY description 수정만이 아니라 코드 자체 검증 강화
- **대안 검토**:
  - description만 수정: 코드 검증 약점 그대로 — Eugene 지적 의미 무시
  - 코드+VERIFY 동시 fix (선택): 검증 강도 실제로 강화. 회귀 위험은 단독 모듈 테스트로 차단
- **선택 이유**: TA-1-01 "필터 초기 상태"는 토글 노출이 아니라 입력값 비어있음 검증이어야. TA-1-23 "TOP10 배지"는 페이지 어디 텍스트가 아니라 카드 자식. TA-2-19 정렬은 reading-order. 모두 코드 자체가 docs 의도를 부족하게 검증 중. **VERIFY는 코드 강도를 드러내는 거울**
- **영향 범위**: TA-1-01/1-23/2-19 spec, 향후 VERIFY 작성 시 "코드가 진짜 검증하는가?" 자가 점검 강제
- **되돌리는 방법**: 강화된 검증이 staging 변동성에 false fail 발생 시 가드 추가 (VERIFY는 유지)
