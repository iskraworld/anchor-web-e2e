# Phase 1 — QA 문서 분석 + 자동화 분류표 생성

> 목표: 11개 QA 문서의 전체 TC-ID를 읽고, 각 케이스를 AUTOMATABLE / MANUAL / SKIP 으로 분류한다.
> 산출물: `docs/anchor-e2e-v2/qa-automation-map.md`
> 예상 소요: 30분

---

## 분류 기준

### AUTOMATABLE (자동화 가능)
Playwright로 실행 가능한 모든 케이스.

| 유형 | 예시 |
|---|---|
| 페이지 접근 / 권한 확인 | 비로그인 상태에서 특정 URL 접근 시 로그인 페이지로 리다이렉트 |
| UI 요소 표시/미표시 | 버튼, 텍스트, 섹션이 화면에 있는지 없는지 |
| 폼 입력 및 저장 | 텍스트 입력, 드롭다운 선택, 저장 후 반영 확인 |
| 에러/유효성 메시지 | 잘못된 입력 시 에러 문구 표시 여부 |
| 네비게이션 | 클릭 후 페이지 이동, URL 변경 |
| 데이터 노출 | 특정 텍스트/숫자/리스트가 화면에 표시되는지 |
| 탭/필터/정렬 | 탭 전환 시 컨텐츠 변경 여부 |

### MANUAL (수동 검증 필요)
AI가 기술적으로 실행 불가능한 케이스. 반드시 이유를 명시한다.

| 유형 | 이유 |
|---|---|
| 이메일 수신 확인 | 외부 메일 서버 — Playwright 접근 불가 |
| 결제/구독 처리 | PG 연동 — 실제 결제 없이 완료 확인 불가 |
| 소셜 로그인 (카카오/구글) | OAuth 팝업 — 자동화 차단됨 |
| PDF 다운로드 내용 검증 | 파일 내용 비교 — 레이아웃 픽셀 수준 판단 필요 |
| SMS 인증 | 외부 통신사 서버 — 수신 확인 불가 |
| 앱 설치 / 딥링크 | 네이티브 앱 영역 |

### SKIP (사람이 수행)
자동화 불가가 아니라, 프로젝트 정책상 사람이 담당하기로 결정된 케이스.

| 유형 | 이유 |
|---|---|
| 신규 회원가입 | 계정 생성 후 삭제 필요 — 스테이징 데이터 오염 방지 |
| 계정 탈퇴 | 계정 삭제 — 복구 불가, 테스트 계정 소실 위험 |
| 관리자 전용 기능 | 관리자 계정 미보유 |

## 작업 절차

### 1단계: QA 문서 읽기

11개 문서를 순서대로 읽는다. 각 문서에서:
- TC-ID 전체 목록 추출
- ~~취소선~~ 처리된 삭제 TC 식별 (목록에서 제외)
- 각 TC의 테스트 절차와 기대결과 확인

### 2단계: 분류 결정

각 TC-ID에 대해 위 기준으로 AUTOMATABLE / MANUAL / SKIP 결정.
판단이 애매한 경우: AUTOMATABLE로 분류하고 Phase 2에서 실제 구현 시도 후 재분류.

### 3단계: qa-automation-map.md 작성

아래 형식으로 `docs/anchor-e2e-v2/qa-automation-map.md` 파일 생성:

```markdown
# QA 자동화 분류표

생성일: YYYY-MM-DD
총 TC: N개 (자동화: N | 수동: N | 스킵: N)

## AUTH — 로그인/회원가입

| TC-ID | 테스트명 | 분류 | 사유 (수동/스킵인 경우) |
|---|---|---|---|
| AUTH-3-01 | 비로그인 접근 시 로그인 리다이렉트 | AUTOMATABLE | |
| AUTH-4-6-01 | 신규 일반 회원가입 | SKIP | 사람이 수행 — 계정 생성 후 삭제 필요 |
| AUTH-4-4-08 | 이메일 인증 링크 수신 확인 | MANUAL | 외부 메일 서버 접근 불가 |
```
