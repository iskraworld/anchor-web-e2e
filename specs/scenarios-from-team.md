## 시나리오 A — 특정 공무원 공통 인맥 탐색

> **사용자 유형:** 전관 세무사 (공무원 출신, 가온세무법인 소속)
> **계정:** bagseongho@gaon.com
> **상황:** 서울청 재산팀 김정희 조사관과의 공통 인맥을 파악해야 한다.

**데이터 의존성**
- 김정희 (현직 공무원 ID `5707`, 서울청 성실납세지원국 소득재산세과 재산팀) 존재
- 박성호 계정과 김정희 사이의 공통 관계망 데이터 (최소 0명 이상)
- 인사권자(김진범 팀장, 최승일 과장) 연결 데이터 (그래프 렌더링 기준)

```gherkin
@P0 @user-tax-official @scenario-A

  Background:
    Given 전관 세무사 박성호(bagseongho@gaon.com)가 로그인되어 있다

  Scenario: A-1 김정희 조사관 검색
    Given 홈의 "현직 공무원 정보 탐색" 탭이 선택되어 있다
    When 공무원명 입력란에 "김정희"를 입력하고 검색 버튼을 클릭한다

    And 검색 결과에 "서울지방국세청" 소속 "김정희" 조사관이 포함된다
    And 해당 항목을 클릭하면 "/search/active-officials/5707"로 이동한다
    # 데이터 의존: 현직 공무원 ID 5707 (김정희, 서울청 소득재산세과 재산팀)

  @P0 @scenario-A-2
  Scenario: A-2 김정희 프로필 및 공통 관계망 UI 확인
    Given "/search/active-officials/5707"에 접근한다
    When 페이지가 로딩된다
    Then 페이지 제목에 "김정희"가 포함된다
    And "공통 관계망 찾기" 섹션이 노출된다
    And "관계망 찾기" 버튼이 존재한다
    # 데이터 의존: ID 5707 공무원 프로필 페이지 존재

  @P0 @scenario-A-3
  Scenario: A-3 나 기준 공통 관계망 조회
    Given "/search/active-officials/5707" 프로필 페이지에 있다
    When "관계망 찾기" 버튼을 클릭한다
    Then 공통 관계망 그래프 또는 결과 영역이 렌더링된다
    And 오류 메시지 없이 결과가 노출된다
    # 데이터 의존: 박성호와 김정희 사이의 관계망 데이터 (직접 관계 없음, 공통 인맥 2명 예상)

  @P0 @scenario-A-4
  Scenario: A-4 인사권자 포함 공통 관계망 조회

    Then 인사권자(김진범 팀장, 최승일 과장)가 포함된 확장 관계망이 렌더링된다
    And 오류 없이 결과가 노출된다
    # 데이터 의존: 김진범(재산팀장), 최승일(소득재산세과장) 노드 존재

    Then 가온세무법인 소속 세무사 전체와 김정희+인사권자 간 공통 인맥 결과가 렌더링된다
    And 오류 없이 결과가 노출된다

    # 고영호 전관이 영향력 1위로 나타나야 함 (박성훈과 동창, 김진범과 동문/옛 상사 관계)

    And 각 인물 항목에 관계 정보(관계 유형 또는 점수)가 표시된다
    # 데이터 의존: 김진범(ID 미확인) 영향력 점수 데이터 존재 필요
```

## 시나리오 C — 세무사 프로필 등록 후 납세자 노출 확인

- 한지희 세무사: taxhan@theanchor.best 계정, 프로필 노출 토글 **OFF** 상태 유지 필요
- 한지희 세무사 UUID: 미확인 (⚠️ Anchor 팀 확인 필요)
