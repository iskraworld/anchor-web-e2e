# QA 자동화 분류표

생성일: 2026-04-28  
총 TC: 840개 (삭제 ~30개 제외 → 유효 ~810개)

---

## 분류 규칙

### AUTOMATABLE — 자동화 가능

| 패턴 유형 | 예시 |
|---|---|
| 접근 권한 확인 | 비로그인/잘못된 계정으로 URL 접근 → 리다이렉트/에러 확인 |
| UI 요소 표시/미표시 | 특정 버튼·텍스트·섹션 존재 여부 |
| 탭·메뉴 네비게이션 | 탭 클릭 → URL 변경 또는 콘텐츠 변경 확인 |
| 폼 유효성 검증 | 잘못된 입력 → 에러 메시지 표시 확인 |
| 데이터 표시 | 특정 텍스트·이름·건수·상태 표시 확인 |
| 필터/검색 동작 | 조건 입력 후 결과 변화 확인 |
| 빈 상태 화면 | 0건 조건 → 빈 상태 안내 표시 |
| 버튼 활성/비활성 | 조건 충족 여부에 따른 버튼 상태 |

### MANUAL — 수동 검증 필요 `[M]`

| 이유 | 해당 TC 유형 |
|---|---|
| 외부 이메일 수신 | 이메일 인증, 이메일 변경 인증 링크 확인 |
| SMS/OTP 수신 | 본인인증, 휴대폰 변경 |
| OAuth 팝업 차단 | 카카오·구글 소셜 로그인 |
| PDF 파일 내용 검증 | PDF 저장 후 내용 육안 확인 |
| 결제/PG 연동 | 구독 결제 처리 완료 |
| 알림 장치 수신 | 외부 알림 도착 여부 |
| 이의신청 검토중 → 승인 상태 전환 | 관리자 액션 필요 (어드민 기능) |
| 파일 검증 처리 결과 | 증빙 파일 검증 완료/반려 상태 (어드민 액션) |

### SKIP — 사람이 수행 `[S]`

| 이유 | 해당 TC 유형 |
|---|---|
| 신규 계정 생성 필요 | 회원가입 플로우 전체 |
| 계정 영구 삭제 위험 | 회원탈퇴, 법인 삭제 |
| 스테이징 데이터 오염 | 법인/팀 신규 생성·등록 |

## MANUAL 항목 목록 (참조용)

각 spec 파일에서 `[M]` 태그로 표기됨. 결과 리포트에서 "수동 검증 필요" 테이블로 분리.

| 참조ID | 모듈 | TC-ID 범위 | 이유 |
|---|---|---|---|
| M-AUTH-01 | AUTH | AUTH-4-5-* (이메일 인증) | 이메일 수신 불가 |
| M-AUTH-02 | AUTH | AUTH-4-6-02, 4-7-02, 4-8-02 (소셜 로그인) | OAuth 팝업 차단 |
| M-AUTH-03 | AUTH | AUTH-4-4-* (SMS 인증 관련) | SMS 수신 불가 |
| M-MY-01 | MY | MY-1-03 (이메일 변경 인증) | 이메일 수신 불가 |
| M-MY-02 | MY | MY-1-08, 1-09 (본인인증/SMS) | SMS 수신 불가 |
| M-SP-01 | SP | 구독 결제 처리 TC | PG 연동 |
| M-EI-01 | EI | 검증 완료/반려/승인 상태 TC | 어드민 액션 필요 |
| M-ER-01 | ER | PDF 다운로드 내용 검증 TC | 파일 내용 육안 확인 |
